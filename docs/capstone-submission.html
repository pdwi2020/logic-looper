Puzzle Game Name

Logic Looper — Daily Puzzle Logic Game

A full-stack offline-first progressive web app delivering one deterministic puzzle per day, with streak tracking, achievement badges, and GitHub-style activity heatmaps.

Section 2

Working Principle

End-to-end deterministic generation, offline persistence, and periodic sync.

HMAC-SHA256 deterministic seed per date → seeded LCG RNG → three puzzle types (Number Matrix / Sequence Solver / Equation Puzzle) → answer validation → IndexedDB save → Redux-triggered batch sync every 5 puzzles
1. Seed generation: Today's date string is hashed with HMAC-SHA256 using crypto-js to produce a deterministic 64-bit hex seed. The same date always generates the same puzzle for every user.
2. RNG: A seeded LCG (Linear Congruential Generator) consumes that seed and produces the puzzle's numbers or sequence values deterministically.
3. Puzzle rendering: Three puzzle families are supported. Number Matrix uses 3×3, 4×4, or 4×4 grids where missing cells are solved through row and column logic. Sequence Solver presents missing elements at a random non-boundary position in arithmetic, geometric, perfect-square, triangular-number, or Fibonacci-inspired sequences. Equation Puzzle displays a styled equation with one missing value (e.g. 3 × ? = 18, (? + 3) × 2 = 14, 5² + ? = 30) across three difficulty tiers.
4. Validation: User input is compared against the computed answer with an O(n) exact-match validation pass.
5. Persistence: A completion record is saved immediately to IndexedDB through idb 8.0, so the core play loop remains fully functional offline.
6. Sync: Redux middleware counts completions, and every fifth completion triggers a batch POST to /api/sync/daily-scores.

Section 3

Features Implemented

Player-facing experience, offline architecture, and progression systems.

Core Product Surface

  • 5 pages: Home, Puzzle Arena, Profile, Leaderboard, Settings
  • 3 puzzle types: Number Matrix, Sequence Solver, and Equation Puzzle
  • 3 difficulty levels each (Easy, Medium, Hard), scaling dynamically every 122 days
  • Responsive UI with mobile hamburger navigation
  • Smooth animations via Framer Motion

Retention and Progression

  • Streak tracking with current streak, longest streak, and an "On Fire" state at 3+ consecutive days
  • GitHub-style 52-week activity heatmap with 5 intensity levels, hover tooltips, and year selector
  • 4 achievement badges: Streak Week, Streak Month, Century, and Perfect Month

Architecture and State

  • Offline-first architecture with IndexedDB as the primary data store and full play without internet
  • Smart batch sync every 5 completions, executed only when the device is online
  • Redux Toolkit global state with typed slices for maintainable data flow

Platform Readiness

  • PWA-ready architecture with service worker, web app manifest, and installability on mobile and desktop
  • Stable user experience in both connected and disconnected scenarios

Section 9

Challenges Faced

Engineering issues encountered during implementation and how they were resolved.

  • Prisma v7 breaking changes: The "url" field was removed from the datasource schema block, requiring migration to environment-variable-only configuration.
  • React controlled components: Input elements using a value prop did not trigger the native change events expected by some test utilities, which was resolved by switching to userEvent from @testing-library/user-event.
  • Achieving 91%+ branch coverage: Vitest branch coverage required explicit tests for offline and online edge cases, error paths, and conditional renders across 19 components.
  • Deterministic seeding: Identical puzzle output across timezones required normalizing date strings to UTC before hashing.
  • IndexedDB in tests: jsdom lacks native IndexedDB support, so fake-indexeddb was required for the database-layer test suite.
